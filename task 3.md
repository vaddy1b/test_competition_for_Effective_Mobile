# Задание 3: Архитектура (Push-уведомления)

## Описание архитектуры

Так как на бэкенде микросервисная архитектура, оптимальным решением является событийно-ориентированная модель (Event-Driven Architecture) с использованием брокера сообщений (Apache Kafka или RabbitMQ).

1. Микросервисы (**Корзина**, **Заказы**, **Маркетинг**) **не отправляют пуши напрямую**. Они генерируют события (Events) и отправляют их в **Message Broker**.
2. **Notification Service** (Микросервис уведомлений) выступает подписчиком (Consumer). Он вычитывает события из брокера, формирует текст сообщения (используя шаблоны и локализацию) и определяет токен устройства пользователя из базы данных.
3. Notification Service отправляет сформированный запрос в облачные провайдеры пушей:
   - для Android — **FCM (Firebase Cloud Messaging)**;
   - для iOS — **APNS (Apple Push Notification Service)**.
4. Мобильное приложение получает уведомление и отображает его.
